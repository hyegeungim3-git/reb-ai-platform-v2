# GenOS 검증 실행

판정은 실행 증거로만 한다. 3단계: 자동 스캔 → 변경 지점 수동 확인 → 빌드.

## 1. 자동 스캔 (항상 먼저)

dev 서버 기동 상태에서:

```
node .claude/skills/genos-verify/scripts/verify.mjs [baseUrl]
```

- 3도메인 × (포털 진입 → 사용자 포털 → GENERAL 스캔 → 에이전트 허브 스캔)
- 판정 항목: 금칙어 0건, 핵심 마커 존재, 시나리오 카드 수, 콘솔 신규 에러 0
- baseUrl 기본값 http://localhost:5173 — 포트가 다르면 dev 서버 시작 로그의 실제 포트를 넘겨라
- 크롬을 못 찾으면 `CHROME_PATH` 환경변수로 chrome.exe 경로 지정
- **EXIT 0 = 전 항목 통과.** 실패 항목은 도메인·항목·원시 값이 출력된다

스크립트가 커버하는 것은 "깨지지 않았는가"다. "새 기능이 동작하는가"는 커버하지 않는다 → 2단계로.

## 2. 변경 지점 수동 확인

- 변경한 바로 그 지점을 실제 조작: 버튼 클릭→상태 변화, 입력→저장→재진입 왕복, 시나리오 실행→완주(릴레이 ~12초 대기), 딥링크→목적지 화면
- DOM 텍스트 판정이 1순위 (스크린샷은 타임아웃이 잦음)
- 자동화 함정(사이드바 접힘, 타이핑 연출 대기, 콘솔 버퍼 누적)은 `../genos-work/references/pitfalls.md` §3 참조
- 반응형 변경이면 375 뷰포트에서 **reload 후** 판정

## 3. 빌드 판정

한글 경로 크래시 때문에 로컬 직접 빌드는 무효다. `docs/QUALITY-CHECKLIST.md` §A의 ASCII 복사 빌드 절차를 따르고(robocopy /XD 절대경로), **EXIT 0 + "✓ built in Xs"** 둘 다 확인 후 복사본을 삭제한다. 또는 push 후 CI로 갈음.

## 배포까지 검증할 때

`gh run watch <id> --exit-status` → 라이브 index 200 → 변경이 든 **해당 청크**에서 마커 문자열 확인 (lazy 컴포넌트는 index-*.js에 없다). Pages deploy만 실패하면 rerun 말고 새 런.
